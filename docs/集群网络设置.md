# 集群网络设置指南

本文档介绍如何在 DGX Spark 集群上配置网络，以支持多节点 vLLM 推理。

## 📋 概述

本指南适用于双节点集群，同样适用于更大的集群。

参考 [此论坛帖子](https://forums.developer.nvidia.com/t/6x-spark-setup/354399/56) 了解 6-8 节点 Spark 集群的配置示例。

**重要提示**: 为了获得最佳的 vLLM 性能，集群节点数量应为 2 的幂次方，例如 2、4 或 8 个节点。

本指南假设节点名称为 `spark` 和 `spark2`，但您可以使用任何名称。
IP 地址同样灵活：我们使用 `192.168.177.0/24` 网段，`.11` 和 `.12` 分配给两个节点，但您可以根据需要分配任何 IP 地址，只要在同一个子网内即可。

---

## 📡 DGX Spark ConnectX 网络特性

DGX Spark 拥有非常独特的 ConnectX 网络配置。

为了实现 200G 传输速度，ConnectX 网卡需要 ~x8 PCIe 5.0 通道。

然而，由于硬件限制，DGX Spark SoC 无法为每个设备提供超过 x4 PCIe 通道。
因此，为了在单根线缆连接上实现 200G 速度，每个物理端口共享同一对 PCIe5 x4 连接。
每个 PCIe 5 x4 链路由两个以太网接口和两个 RoCE 接口表示：

```bash
eugr@spark:~$ ibdev2netdev
rocep1s0f0 port 1 ==> enp1s0f0np0 (Down)
rocep1s0f1 port 1 ==> enp1s0f1np1 (Up)
roceP2p1s0f0 port 1 ==> enP2p1s0f0np0 (Down)
roceP2p1s0f1 port 1 ==> enP2p1s0f1np1 (Up)
```

在这种情况下，单根线缆连接到最外侧的 QSFP 端口（从背面看，是右边的端口）。
该端口有两个配对的"双胞胎"接口：

- **以太网**: `enp1s0f1np1` 和 `enP2p1s0f1np1`
- **RoCE/IB**: `rocep1s0f1` 和 `roceP2p1s0f1`

每个"双胞胎"接口代表一个 PCIe x4 链路，可提供高达 100G 的链路速度。

对于 vLLM，我们需要 RDMA over RoCE，因此以太网速度不是那么重要，我们只需要给其中一个接口分配 IP 地址即可 - 本例中为 `enp1s0f1np1`。
然而，为了在 NCCL RDMA 模式下获得全部带宽，我们需要同时使用**两个**RoCE"双胞胎"接口。这是通过将 `NCCL_IB_HCA` 设置为两个 RoCE 接口实现的：`export NCCL_IB_HCA=rocep1s0f1,roceP2p1s0f1`

`./launch-cluster.sh` 会自动执行此操作，以及自动检测接口，因此只要您正确设置了以太网接口，vLLM 就会利用两个 RoCE"双胞胎"接口。

另外，请注意，使用**两个**端口连接两个 Spark 不会带来明显的带宽优势，单连接就足够了。
如果您通过级联方式连接 3 个 Spark，您只能在每对 Spark 之间维持 100G 速度。

## 🌐 网络配置

假设两个节点都使用最右侧的 QSFP 端口连接（从背面看）。

### 配置 Spark 节点 1

在 `spark` 节点上创建 `/etc/netplan/40-cx7.yaml`：

```yaml
network:
  version: 2
  ethernets:
    enp1s0f1np1:
      dhcp4: no
      dhcp6: no        # 显式禁用 DHCPv6
      link-local: []   # 限制链路本地地址仅为静态 IPv4
      mtu: 9000
      addresses: [192.168.177.11/24]
    enP2p1s0f1np1:
      dhcp4: no
      dhcp6: no
      link-local: [ ipv4 ]
      mtu: 9000
```

### 配置 Spark 节点 2

在 `spark2` 节点上创建 `/etc/netplan/40-cx7.yaml`：

```yaml
network:
  version: 2
  ethernets:
    enp1s0f1np1:
      dhcp4: no
      dhcp6: no        # 显式禁用 DHCPv6
      link-local: []   # 限制链路本地地址仅为静态 IPv4
      mtu: 9000
      addresses: [192.168.177.12/24]
    enP2p1s0f1np1:
      dhcp4: no
      dhcp6: no
      link-local: [ ipv4 ]
      mtu: 9000
```

**注意**: 在"双胞胎"接口对中，只需要给一个接口分配 IP 地址，但两个接口都需要设置 MTU。
如果您希望独立使用第二个接口，也可以为另一个"双胞胎"接口分配单独的地址，但请确保使用不同子网的 IP 地址。

例如，在上述示例中，如果您想为 `enP2p1s0f1np1` 分配 IP，需要在 `spark` 上使用 `192.168.177.11`。**不要对两个"双胞胎"接口使用相同的子网** - 这会混淆自动发现并打乱路由。

这不会影响 vLLM 性能，因为它会使用 RDMA over RoCE 利用两个"双胞胎"接口，即使 IP 只设置在一个上。

### 应用网络配置

然后在每个节点上运行：

```bash
sudo chmod 600 /etc/netplan/40-cx7.yaml
sudo netplan apply
```

### 设置免密 SSH

在 `spark` 节点上：

```bash
wget https://raw.githubusercontent.com/NVIDIA/dgx-spark-playbooks/refs/heads/main/nvidia/connect-two-sparks/assets/discover-sparks
chmod +x discover-sparks
./discover-sparks
```

### 测试 MTU 设置

```bash
sudo ip link set dev enp1s0f1np1 mtu 9000
```

## 🔧 NCCL 配置

参考：https://build.nvidia.com/spark/nccl/stacked-sparks

### 安装和构建 NCCL

```bash
# 安装依赖并构建 NCCL
sudo apt-get update && sudo apt-get install -y libopenmpi-dev
git clone -b v2.28.3-1 https://github.com/NVIDIA/nccl.git ~/nccl/
cd ~/nccl/
make -j src.build NVCC_GENCODE="-gencode=arch=compute_121,code=sm_121"

# 设置环境变量
export CUDA_HOME="/usr/local/cuda"
export MPI_HOME="/usr/lib/aarch64-linux-gnu/openmpi"
export NCCL_HOME="$HOME/nccl/build/"
export LD_LIBRARY_PATH="$NCCL_HOME/lib:$CUDA_HOME/lib64/:$MPI_HOME/lib:$LD_LIBRARY_PATH"
```

### 构建 NCCL 测试套件

```bash
# 克隆并构建 NCCL 测试
git clone https://github.com/NVIDIA/nccl-tests.git ~/nccl-tests/
cd ~/nccl-tests/
make MPI=1
```

### 在两个节点上测试

```bash
# 设置网络接口环境变量（使用您的活动接口）
export UCX_NET_DEVICES=enp1s0f1np1
export NCCL_SOCKET_IFNAME=enp1s0f1np1
export OMPI_MCA_btl_tcp_if_include=enp1s0f1np1
export NCCL_IB_HCA=rocep1s0f1,roceP2p1s0f1
export NCCL_IB_DISABLE=0

# 在两个节点上运行 all_gather 性能测试
mpirun -np 2 -H 192.168.177.11:1,192.168.177.12:1 \
  --mca plm_rsh_agent "ssh -o UserKnownHostsFile=/dev/null -o StrictHostKeyChecking=no" \
  -x LD_LIBRARY_PATH=$LD_LIBRARY_PATH \
  $HOME/nccl-tests/build/all_gather_perf -b 16G -e 16G -f 2
```

## 📝 关键配置总结

| 配置项 | 值 | 说明 |
|--------|------|------|
| **MTU** | 9000 | 设置 jumbo frames |
| **NCCL_IB_HCA** | `rocep1s0f1,roceP2p1s0f1` | 同时使用两个 RoCE 接口 |
| **NCCL_IB_DISABLE** | `0` | 启用 RDMA over RoCE |
| **子网** | 同一网段 | 确保所有节点在同一子网 |
| **IP 分配** | 每个接口一个 | 不要对"双胞胎"接口使用相同子网 |

## 🔍 故障排查

### 如果自动发现失败

确保：
1. 所有节点在同一网段
2. 防火墙允许 SSH 和 NCCL 通信
3. 正确配置了免密 SSH

### 如果带宽不达标

检查：
1. 两个 RoCE 接口是否都正确配置
2. MTU 设置是否正确
3. 物理连接是否稳定

### 如果延迟过高

检查：
1. 网络接口是否正常工作
2. 是否有其他进程占用带宽
3. 物理连接质量
